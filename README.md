# 1 Step: Crear el folder donde contruiremos nuetro proyecto

# 2 Step: Crear el project con comando: npm init. Automaticamente se crea el package.json

# 3 Step: crear el .gitignore para ignorar el node_modules al subir proyecto al repo git

# 4 Step: Descargar dependencias para el front con react: npm i react y npm i react-dom y npm i react-router-dom 

# 5 Step: Descargar dependencias para el back con node: npm i express

# 6 Step: Descargar dependencias de desarrollo para el back: npm i nodemon -D

# NOTA: nodemon se debe incluir en el script "start" para recargar los cambios automaticamente

# 7 Step: Crear sobre el main root, el file: webpack.config.js

# 8 Step: Descargr dependencias de desarrollo para el webpack:

# - npm i webpack -D "El núcleo de la herramienta, se encarga de la construcción de los bundles"

# - npm i webpack-cli -D "Proporciona la interfaz de línea de comandos para ejecutar Webpack."

# - npm i webpack-dev-server -D recarga la página automáticamente cada vez que se guarda un cambio.

# - npm install --save-dev @babel/core @babel/preset-env @babel/preset-react babel-loader

# Babel: Si utilizamos sintaxis de JavaScript más moderna (ES6+, JSX), necesitaremos Babel para transpilarla a código compatible con navegadores más antiguos.

# npm install --save-dev css-loader style-loader file-loader url-loader

# Loaders: Son plugins que permiten a Webpack procesar diferentes tipos de archivos (CSS, imágenes, etc.). Algunos loaders comunes son:

# babel-loader: Para transpilar código JavaScript con Babel.

# css-loader, style-loader: Para cargar y aplicar estilos CSS.

# file-loader, url-loader: Para manejar archivos estáticos como imágenes y

# npm i html-webpack-plugin -D

# npm i clean-webpack-plugin -D

# Plugins: Amplían la funcionalidad de Webpack. Algunos plugins útiles son: html-webpack-plugin: Para generar archivos HTML que incluyen los bundles de JavaScript.
